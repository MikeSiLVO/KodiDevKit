# Skin Shortcuts integration

## What is Skin Shortcuts?

[script.skinshortcuts](https://github.com/MikeSiLVO/script.skinshortcuts) is a Kodi addon that generates menu XML at runtime. Skins define a template file in `shortcuts/`, and the addon fills it with user-configured menu entries at startup, producing files like `script-skinshortcuts-includes.xml`.

This means some includes, variables, and XML structures only exist at runtime and cannot be statically validated. KodiDevKit detects Skin Shortcuts usage and adjusts validation accordingly.

## Detection

KodiDevKit checks for Skin Shortcuts in two ways:

1. **addon.xml dependency** -- Parses `addon.xml` for `<import addon="script.skinshortcuts"/>`. If found, runtime include patterns are suppressed during validation.

2. **Template file scanning** -- Looks for template files in `shortcuts/` to find static references to skin includes and variables.

## v2 vs v3

| | v2 | v3 |
|---|---|---|
| Template file | `shortcuts/template.xml` | `shortcuts/templates.xml` |
| Override files | `shortcuts/overrides.xml` | Split across multiple files |
| Include syntax | `<include>Name</include>` | `<include content="Name">` and `<skinshortcuts include="Name">` |

KodiDevKit checks for v3 first (`templates.xml`), then falls back to v2 (`template.xml`).

## What KodiDevKit marks as used

When scanning template files, these references count as "used" (preventing false unused reports):

- **`$VAR[Name]` / `$ESCVAR[Name]`** in text content and attribute values -- marks the variable as used.
- **`<include>Name</include>`** tags -- marks the include as used.
- **`content="Name"` and `include="Name"` attributes** (v3) -- marks the include as used.

## What KodiDevKit skips

These are runtime placeholders that cannot be resolved statically:

- **`$SKINSHORTCUTS[...]`** -- Skin Shortcuts runtime values (menu labels, paths, etc.).
- **`$PROPERTY[...]`** -- Dynamic property values set at runtime.

Any variable or include reference containing these placeholders is ignored during validation.

## Runtime include patterns suppressed

When `script.skinshortcuts` is detected as a dependency, include references matching these patterns are not flagged as "undefined":

| Pattern | Example | Source |
|---------|---------|--------|
| `skinshortcuts-*` | `skinshortcuts-main-menu` | Generated by script.skinshortcuts |
| `[digits]menu` | `0menu`, `3menu` | SkinShortcuts menu IDs |
| `script-*-includes` | `script-skinshortcuts-includes` | Dynamic script includes |

Similarly, when `script.skinvariables` is a dependency:

| Pattern | Example | Source |
|---------|---------|--------|
| `skinvariables-*` | `skinvariables-colorset` | Generated by script.skinvariables |

## Generated file handling

`script-skinshortcuts-includes.xml` receives special handling:

- **Include loading** -- The file is excluded from recursive include file loading (it doesn't exist at development time).
- **Label/ID validation** -- Content from this file is excluded since it's entirely generated at runtime.

## Excluded from validation

The `shortcuts/` folder is in the default [`validation_exclude`](settings.md#validation) list, so template files themselves are not validated for XML structure issues.
